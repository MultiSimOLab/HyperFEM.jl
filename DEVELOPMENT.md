
## Administrators

### Publishing a new release

1. Update the project version in `Project.toml`. Make a PR and merge it, once the tests have passed.
2. Open an issue or go to the previous commit. Create a comment and type the following:
```
@JuliaRegistrator register
```

The registrator will open a PR in the Julia registry. Once the package is published, the tag bot will create a release and generate the changelog. The documentation will be updated as well.

#### ⚠️ Breaking changes (increase first non-zero digit, e.g. `0.0.1 -> 0.0.2`)

If so, you will need to tell the registrator the list of breaking changes:
```
@JuliaRegistrator register

Release notes:

## Breaking changes

- blah
```


### TagBot

Thanks to this action, the changelog file is not required. However, the tag bot needs special write permisions, so make sure the `ssh-key` granting permissions to tag bot is valid within the repo settings.
You may need the package `DocumenterTools` to create the `sss-key` for the tag bot.


### Documentation

Documentation is generated by `docs/make.jl` and triggered by `workflows/docs.yml` after a PR is closed (dev) or a tag is created (stable).
